## 1. 문제 정의

**서비스명:** MoodTrack  
**한 줄 소개:** 지금 기분을 입력하면 AI가 감정을 분석해 플레이리스트를 즉시 추천해주는 웹 서비스

**문제 상황**
- 기존 스트리밍 서비스는 과거 청취 이력 기반 추천만 제공
- 사용자가 현재 감정에 맞는 음악을 찾는 데 평균 4~7분 소요
- 복합적인 감정을 키워드로 변환해야 하는 인지 부담 존재

**해결 방안**
- 자연어로 감정 입력 → LLM이 감정 벡터로 분석 → Spotify/YouTube 플레이리스트 3개 즉시 추천

## 2. 페르소나

**이민재 (27세, 대학원생)**
- 서울 소재 대학원 심리학과 석사 2년차
- 하루 평균 음악 청취 3~4시간, Spotify 유료 구독
- 감정 기복이 잦고 음악으로 감정을 조절하려는 경향

**페인포인트**
- 논문 작업 중 기분에 맞는 음악 찾느라 5분 이상 낭비
- Spotify 추천이 과거 이력 기반이라 현재 감정 미반영
- 복합 감정을 검색 키워드로 변환하는 것 자체가 스트레스

**니즈**
- 감정을 말로 설명하면 5초 이내 플레이리스트 추천
- 복잡한 감정도 자연어 그대로 입력 가능
- 새로운 아티스트/장르 발견 가능한 큐레이션

## 3. 시나리오

**시나리오 1 - 감정 입력 후 플레이리스트 추천**
- 도입: 밤 11시, 논문 작업 중 집중력이 끊김
- 계기: 친구에게 MoodTrack 링크 공유받아 접속
- 행동 1: 입력창에 "논문 막혀서 답답하고 멍하니 있고 싶어" 입력
- 행동 2: 5초 후 감정 분석 결과 및 플레이리스트 3개 표시
- 결과: Spotify로 이동해 음악 재생, 논문 작업 재개
- Unhappy Path: 링크 오류 시 "다시 추천받기" 버튼 제공

## 4. IA (정보 구조)

```
MoodTrack
├── 홈
│   ├── 감정 입력창
│   └── 최근 기록 미리보기
├── 추천 결과
│   ├── 감정 분석 결과
│   ├── 플레이리스트 목록
│   └── 플레이리스트 상세
├── 감정 히스토리
│   ├── 날짜별 목록
│   ├── 상세 보기
│   └── 빈 상태 (기록 없음)
├── 설정
│   └── 계정 관리
└── 예외 화면
    ├── 에러 페이지
    └── 로딩 화면
```

## 5. 사용자 플로우

**시나리오 1 - 감정 입력 후 추천**

시작 → 홈 화면 접속 → 감정 텍스트 입력 → 입력값 있는가?  
- Yes: 분석 요청 → AI 감정 분석 → 분석 성공?
  - Yes: 추천 결과 표시 → 플레이리스트 클릭 → YouTube 이동 → 종료
  - No: 에러 페이지 표시 → 다시 시도
- No: 경고 메시지 표시 → 다시 입력

**시나리오 2 - 감정 히스토리 조회**

시작 → 히스토리 메뉴 클릭 → 기록 있는가?  
- Yes: 날짜별 목록 표시 → 날짜 클릭 → 상세 기록 표시 → 종료
- No: 빈 상태 화면 표시 → 홈으로 이동

**핵심 의사결정 근거**
- 홈 화면 중앙에 입력창 배치: 서비스 핵심 기능에 즉시 접근 가능하도록
